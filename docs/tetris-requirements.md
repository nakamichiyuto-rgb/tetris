# テトリスWEBアプリ 要件定義書

## 1. 文書情報

- 文書名: テトリスWEBアプリ 要件定義書
- 作成日: 2026-04-21
- 対象: ローカル環境で動作するブラウザ版テトリス
- 目的: 開発に着手するための要件を整理し、実装範囲と品質基準を明確化する

## 2. 背景と目的

本アプリは、PCブラウザ上でローカル実行できるシンプルなテトリスゲームを提供することを目的とする。
インターネット接続やサーバー構築を前提とせず、開発者または利用者がローカル環境で起動し、すぐに遊べる構成とする。

## 3. スコープ

### 3.1 対象範囲

- ブラウザ上で動作するシングルプレイのテトリスゲーム
- ローカルPCでの起動とプレイ
- スコア表示
- レベルアップ
- ライン消去
- ゲームオーバー判定
- キーボード操作
- リスタート機能

### 3.2 対象外

- オンライン対戦
- ユーザー登録、ログイン
- サーバーサイドAPI
- 課金機能
- SNS共有
- リアルタイムランキング連携

## 4. 想定利用者

- ローカル環境で気軽に遊びたい個人ユーザー
- フロントエンド学習用にゲーム実装を試したい開発者
- テトリスの基本ロジックを確認したい制作者

## 5. 利用環境

### 5.1 実行環境

- OS: macOS / Windows / Linux
- ブラウザ: Google Chrome, Microsoft Edge, Safari, Firefox の主要最新版
- ネットワーク: 不要

### 5.2 開発環境

- ローカルPC上で起動可能であること
- Node.js ベースの開発環境を想定してよい
- 実装技術は HTML / CSS / JavaScript を基本とし、必要に応じて TypeScript やフレームワークを採用可能

## 6. システム概要

本システムは、単一ページで動作するフロントエンドアプリケーションとする。
ゲーム盤面、現在スコア、レベル、次のブロック、ゲーム状態を画面上に表示し、キーボード操作に応じてゲーム状態を更新する。

## 7. 機能要件

### 7.1 ゲーム開始機能

- アプリ起動時にゲーム画面を表示すること
- 初回表示時にゲーム開始可能な状態であること
- 必要に応じて「開始」または「リスタート」操作でゲームを開始できること

### 7.2 ブロック生成機能

- テトリミノ7種（I, O, T, S, Z, J, L）を生成すること
- 生成位置は盤面上部の初期位置とすること
- ブロック出現時に他ブロックと重なっている場合はゲームオーバーとすること

### 7.3 ブロック操作機能

- 左移動ができること
- 右移動ができること
- 回転ができること
- 下移動ができること
- ハードドロップができること
- 壁や積み上がったブロックに衝突する場合は不正な移動を無効化すること

### 7.4 自動落下機能

- 一定時間ごとにブロックが1マスずつ自動落下すること
- レベル上昇に応じて落下速度が上がること

### 7.5 着地判定機能

- ブロックが床または他ブロックに接触した場合、着地判定を行うこと
- 着地後は盤面に固定されること
- 固定後に次ブロックを生成すること

### 7.6 ライン消去機能

- 横一列が隙間なく埋まった場合、そのラインを消去すること
- 同時に複数ラインが揃った場合はまとめて消去できること
- ライン消去後、上部のブロックが落下して詰められること

### 7.7 スコア機能

- ライン消去数に応じてスコアを加算すること
- 同時消去ライン数に応じて加点量を変えること
- 現在スコアを常時表示すること

### 7.8 レベル機能

- 一定条件でレベルアップすること
- 現在レベルを表示すること
- レベル上昇に伴い難易度が上がること

### 7.9 次ブロック表示機能

- 次に出現するブロックを表示すること
- 表示位置は盤面外のサイドパネルとすること

### 7.10 一時停止機能

- プレイ中に一時停止できること
- 一時停止中はブロックの自動落下と入力処理を停止すること
- 再開操作で元の状態から続行できること

### 7.11 ゲームオーバー機能

- 新規ブロックを正常に配置できない場合、ゲームオーバーとすること
- ゲームオーバー時に結果が分かる表示を行うこと
- リスタートにより新規ゲームを開始できること

### 7.12 操作説明表示機能

- 利用者が操作方法を確認できること
- 主要キー操作を画面内に表示すること

## 8. 画面要件

### 8.1 画面構成

- ゲーム盤面
- スコア表示エリア
- レベル表示エリア
- 次ブロック表示エリア
- 操作説明エリア
- 開始 / 一時停止 / リスタート操作エリア

### 8.2 UI要件

- 盤面サイズは視認性を考慮した固定またはレスポンシブ構成とする
- ブロックの種類ごとに識別可能な色を設定する
- ゲーム状態（プレイ中、一時停止、ゲームオーバー）が分かりやすく表示されること
- PCブラウザで快適に操作できること

## 9. 操作要件

以下のキー操作を標準案とする。

- `←`: 左移動
- `→`: 右移動
- `↓`: ソフトドロップ
- `↑` または `X`: 回転
- `Space`: ハードドロップ
- `P`: 一時停止 / 再開
- `R`: リスタート

※ 実装時にキー割り当ては調整可能だが、画面表示と仕様を一致させること。

## 10. データ要件

### 10.1 管理対象データ

- 盤面の状態
- 現在操作中のブロック情報
- 次ブロック情報
- スコア
- レベル
- 消去ライン数
- ゲーム状態（未開始、プレイ中、一時停止、ゲームオーバー）

### 10.2 データ保存

- MVPでは永続化は必須としない
- 拡張要件としてハイスコアのローカル保存（LocalStorage）を検討可能

## 11. 非機能要件

### 11.1 性能

- 一般的なノートPCで滑らかに動作すること
- 入力操作に対する画面反応が体感上遅延しないこと
- ゲームループは安定して動作すること

### 11.2 保守性

- ゲームロジック、描画、入力処理を分離した設計とすること
- 定数やスコア計算ルールは変更しやすい構造とすること

### 11.3 可読性

- 主要処理には適切な命名を行うこと
- 複雑なゲームロジックには必要最小限のコメントを付与すること

### 11.4 可用性

- ローカル環境で安定して起動できること
- ブラウザ再読込時もアプリが破綻しないこと

### 11.5 拡張性

- 将来的に以下の追加がしやすい構成とすること
- ハイスコア保存
- 効果音
- ホールド機能
- ゴースト表示
- スマートフォン対応
- オンラインランキング

## 12. 制約条件

- ローカルWEBアプリとして動作すること
- サーバー必須構成にしないこと
- 著作権上問題のない独自アセットまたは汎用表現を使用すること
- 主要ブラウザで大きな表示崩れがないこと

## 13. 受け入れ条件

- ブラウザ上でゲームを起動できること
- ブロック7種が正常に出現すること
- 左右移動、回転、落下が行えること
- ライン消去が正しく行われること
- スコアとレベルが表示され、条件に応じて更新されること
- 一時停止と再開ができること
- ゲームオーバー判定が正しく動作すること
- リスタート後に初期状態から再プレイできること
